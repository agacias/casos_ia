Auditoría de Código

Meta-prompt

dame un prompt para hacer una revisión de código exhaustiva

Prompt

"Actúa como un **Principal Software Engineer** obsesionado con la **calidad de código**. Realiza una Code Review detallada del siguiente fragmento o archivo. Tu objetivo es asegurar que el código sea limpio, mantenible y siga los principios SOLID."

Evalúa el código bajo los siguientes criterios:

  • Legibilidad y Naming: ¿Las variables y funciones tienen nombres descriptivos? ¿El código es autodocumentado?
  • Complejidad: Identifica funciones demasiado largas o complejas (Complejidad Ciclomática alta).
  • DRY (Don't Repeat Yourself): Detecta lógica duplicada.
  • Mejores Prácticas: Verifica el uso correcto de patrones de diseño y características modernas del lenguaje.
  • Manejo de Errores: Comprueba que las excepciones se capturan y manejan adecuadamente.

Ejemplo de Output Estructurado

Pide a la IA que estructure la revisión así:

Categoría Ubicación Sugerencia Severidad
Naming auth.js:L23 Renombrar `x` a `userToken` para mayor claridad. Baja
DRY utils.js:L45-60 Esta lógica de formateo de fechas se repite en 3 archivos. Extraer a helper `formatDate()`. Media
Error Handling api.js:L10 El bloque try/catch está vacío. Loggear el error o relanzarlo. Alta

Prompts Adicionales

  • "Moderniza este código": "Reescribe este código 'legacy' utilizando las características más nuevas de [Lenguaje, ej. ES6+, Python 3.12]."
  • "Optimiza la legibilidad": "Simplifica las condicionales anidadas (Guard Clauses) y extrae subfunciones para que este método se lea como prosa."
