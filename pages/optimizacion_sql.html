Optimización SQL

Meta-prompt

Optimiza esta consulta SQL lenta y sugiéreme índices.

Prompt

"Actúa como un **DBA Senior** experto en PostgreSQL (o MySQL). Analiza la siguiente consulta que está tardando 3 segundos en ejecutarse sobre una tabla de 5 millones de registros. Tu objetivo es reducir el tiempo de ejecución a milisegundos."

SELECT u.name, count(o.id) as orders
FROM users u
LEFT JOIN orders o ON u.id = o.user_id
WHERE u.created_at > '2023-01-01'
GROUP BY u.name
HAVING count(o.id) > 5;

Proporciona:

  • Análisis del EXPLAIN: ¿Por qué es lenta? (Full Table Scan, Filesort, etc.)
  • Query Refactorizada: Reescribe la consulta si es posible (ej. eliminar subqueries innecesarias).
  • Índices Sugeridos: Comandos exactos para crear los índices necesarios.

Ejemplo de Output Estructurado

Acción Detalle Impacto Estimado
Índice Compuesto `CREATE INDEX idx_users_date ON users(created_at, id)` Alto (Evita Full Scan)
Refactoring Evitar `LEFT JOIN` si solo buscamos usuarios con órdenes. Usar `INNER JOIN`. Medio
